# Perplexity / Gemini / Grok ディスカッションまとめ
(深夜眠れない人向けウェブサービス「Night Lantern Stroll (仮)」)

このドキュメントは、3つのAI（Perplexity → Gemini → Grok）との会話で出てきた主要トピックを時系列・論点ごとに整理したものです。

## 1. Perplexityパート（初期コンセプト固め）
- 核心コンセプトの定義
  - 「話すこと」より「同じ時間に起きている他者の気配」を感じて安心する
  - 「深夜にだけうっすら灯る灯台」 / 「静かな待合室」
- 深夜限定の仕組み
  - 位置情報はタイムゾーンレベル or ユーザー申告の「夜の時間帯」（23:00–5:00）のみ
  - 位置データ即破棄でプライバシー重視
- インタラクションの最小化
  - 発言なしでも「いま起きている人数」「シルエットが流れる」だけで安心感
  - スタンプ限定（文字チャットなし or 極短制限）
  - 既読気配 / うなずきアニメーション程度
- アッパー/ダウナー分岐
  - 「今の気分」アイコン選択で部屋分け（静かに落ち着きたい / ただ寂しい / ハイテンションなど）
  - UIで雰囲気差（明るさ・色調）
- 出会い系防止の徹底仕様
  - 完全ゲスト / ランダム名 / 毎回自動ニックネーム
  - DM・プロフィール・固定ハンドルなし
  - 連絡先交換ワード自動マスク
- 環境音の提案
  - 夜の街の微かな音（新聞配達バイク、風、足音）
  - ホワイトノイズ / 睡眠アプリ寄り
- 完全ゲストのメリット・デメリット議論
  - メリット: 即入室・気軽・プライバシー最高
  - デメリット: 荒らしリスク・常連気配薄い → AIボットで補完提案
- 類似アプリ調査
  - Night Owl / SilentSanctum / HiddenHub / Calm など
  - 「スタンプのみ深夜限定」は未開拓ニッチ
- 収益化初期案
  - プレミアム音パック / スタンプテーマ / 睡眠関連スポンサー

## 2. Geminiパート（世界観深化・UI具体化）
- 世界観の統一
  - 「眠れない夜に家を出て夜の街を散歩する」イメージ
  - ユーザーは「夜の散歩中」の状態
- 画面構成のラフ設計
  - 中央に水平の「夜の道」（時間軸プログレスバー: 左23:00 → 右05:00）
  - 自分のアイコンがゆっくり右へ進む
  - 距離感演出: 近い時刻の人ははっきり、遠い人は霞む
- 広告の自然な統合
  - 道の脇に自動販売機・コンビニ看板・ビル屋上看板
  - 深夜進むと明るさ・色が暗くなる
  - タップで「夜に優しい情報ページ」や音サンプルへ（即商品ページ禁止）
- インタラクションの最終詰め
  - 5文字チャットリスク議論 → 定型選択式を推奨
  - スタンプ＋定型文で「会話にならない」距離感を維持
- その他
  - AIボットで常連感演出
  - スポット音（バイク通過などステレオ演出）

## 3. Grokパート（現実落とし込み・MVP仕様固め）
- PWA採用の推奨
  - インストール不要・ホーム画面追加・オフライン一部対応・プッシュ通知可能
  - ネイティブより低コスト・即公開可能
- インタラクション最終決定
  - スタンプ（20種）＋定型5文字選択式（30種）
  - 「…」連打で隠し羊カウント（5回トリガー、自分だけ見える、時間経過でフェードアウト）
- 環境音詳細
  - MVP: 「夜の街」1テーマ
  - 時間帯自動変化（23時: やや賑やか → 4時: ほぼ静寂）
  - 後で拡張: 海・田舎テーマ選択
- 広告の低コスト・低鬱陶実装
  - 風景の一部として控えめ配置
  - タップ先は静的情報ページ（スポンサー素材埋め込み）
- 将来実装アイデア整理
  - 羊カウント拡張 / 季節スタンプ / 残り香機能 / AI個性（街灯くんなど） / 夜明け演出 / マイ散歩ログ（オプトイン） など
- Firebaseデータ構造案
  - active_users / stamps_log / ai_patterns / config
- タイトル案集約
  - Night Lantern Stroll / Quiet Night Stroll / Midnight Lantern Walk
  - 日本語: 夜灯散歩 / 深夜の灯り散歩 など（Night入り重視、重複回避）
- その他現実評価
  - 完全ゲストメリット大、荒らしは制限＋AI検知でカバー
  - 再遭遇回避（人間同士はランダムが自然）
  - マイ散歩ログはMVP不要、後で検討
