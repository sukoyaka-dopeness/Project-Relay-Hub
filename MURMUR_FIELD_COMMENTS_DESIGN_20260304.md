# 1. コメント種別の定義

コメントは投稿時に種別を選択する。**種別は投稿後に変更できない**（後述）。

| 種別 | 表示名 | 到達範囲 | 備考 |
|---|---|---|---|
| 完全私的 | 「ひとりごと」 | 自分のみ | 外から一切見えない |
| クラスタ向け | 「つぶやき」 | 同クラスタ・距離減衰あり | 近いほど明瞭、遠いほど霧 |
| コンテンツ添付 | 「引用メモ」 | コンテンツに紐づく・距離減衰あり | 返信不可・集合非表示 |

---

# 2. 投稿後の種別変更ポリシー

**コメントは書いた時点の公開設定で固定する。**

- 「ひとりごと」→「つぶやき」への変換は、個別に明示的な操作が必要。
- 設定をグローバルに変更しても、**過去のコメントには遡及しない**。
- 理由：意図せず「ひとりごと」が公開される事故を防ぐため。安心感がサービスの根幹。

## コメントごとの種別選択

ブックマーク時のUI上で、コメントごとに「ひとりごと／つぶやき／引用メモ」を選択できる。
デフォルトはユーザー設定に従うが、投稿単位でオーバーライド可能。

# 3. デフォルト設定とオンボーディング

## オンボーディング猶予期間（検証対象仮説）

> 他サービスに前例のない設計。ベータで検証が必要。

- 登録後 **最初の30日間**、全コメントが「ひとりごと」に強制固定。
- 31日目以降に「つぶやきを許可する」スイッチが現れる。
- 意図：まず安全に慣れてから、自発的に勇気を出す設計。

**検証項目：**
- 猶予期間後に「つぶやき」に移行するユーザーの割合
- 猶予期間中の定着率
- 猶予期間の長さの最適値（30日が適切か）

# 4. 連鎖フィードバック（数字を出さない通知設計）

## 原則
- 「◯人に届きました」という数値通知は出さない。
- 「誰に届いたか」も出さない。
- **気配と波紋だけを返す。**

## 実装案

### 文言による通知
> 「あなたのつぶやきから、誰かが同じ記事に向かった気配があります」

- 人数なし、誰かも不明。
- 「気配」という言語がMURMUR FIELDの文化を形成する。

### 波紋アニメーション
- 自分のブックマークカードに、小さく波紋が広がるアニメーションが一瞬だけ出る。
- 回数・人数は表示しない。「何かが起きた」という事実だけ伝える。
- **「誰かに伝わったという意味だ」とユーザーが理解できる**ことが必須。
- オンボーディング時に「波紋の意味」を一度だけ説明する。

# 5. 返信・集合の非生成

- 返信機能なし
- コメント集合ページなし
- 「◯件のコメント」表示なし
- 引用コメントは対象コンテンツに紐づくが、集合として見せない

# 7. UIの楽しみ（spec追記用メモ）

現行specに薄い部分として、以下をPhase設計に組み込む。

| 項目 | プラットフォーム | Phase目安 |
|---|---|---|
| キーバインドナビゲーション（J/Kで移動等） | PC | Phase 0 |
| ブックマーク時の触覚フィードバック（haptic） | スマートフォン | Phase 0 |
| 波紋アニメーション（連鎖フィードバック） | 両方 | Phase 0 |
| 星座・菌糸ビュー（自クラスタの可視化） | 両方 | Phase 1 |
| 月次フィードバックのビジュアル表現 | 両方 | Phase 1 |

**触る楽しみ・UIのリズムが離脱を防ぐ。** Tumblrのキーバインド、RSSリーダーのサクサク感は「読む行為にリズムが生まれる」体験。スマートフォンのhapticはブックマークという行為を儀式化する。
