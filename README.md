# Project-Relay-Hub

Multi-AI リレーのための **プロジェクト横断ハブ**。  
複数の物語・技術・研究プロジェクトを、AI と人間が協力して継続的に発展させるための **引き継ぎ文書・圧縮仕様・テンプレート** を一元管理するリポジトリです。

このリポジトリは、AI セッション間の「記憶の継承」「構造の維持」「誤解の防止」を目的とした **中枢ステーション（Relay Hub）** として機能します。

---

## 🌐 Purpose / 目的

- 複数プロジェクトの **AIリレー（AI → AI → AI）** を安全に継続させる  
- 各プロジェクトの **仕様・哲学・構造** を圧縮形式で保持  
- AI が誤解しないための **引き継ぎテンプレート** を統一  
- プロジェクトごとの **MASTER_HANDBOOK（完全版仕様書）** を格納  
- 3000年スケールで持続する **文化・構造・物語の保全**

## 🔧 What This Hub Solves / このハブが解決する問題

### 1. **AI の記憶喪失問題**
無料AIはセッションを跨ぐと記憶を保持できないため、  
**構造・仕様・哲学を毎回ゼロから説明するコスト**が発生する。

→ このハブは **圧縮仕様（COMPRESSED）** と **サンプル（SAMPLES）** を提供し、  
次のAIが即座にプロジェクトの文脈を理解できるようにする。

---

### 2. **プロジェクトごとの仕様の分散問題**
複数プロジェクトを並行で扱うと、  
仕様書・テンプレート・サンプルが散らばり、管理が困難になる。

→ このハブは **プロジェクトごとにディレクトリを分けて統合管理**する。

---

### 3. **AI間の誤解・破綻問題**
AIごとに解釈が異なるため、  
物語構造や技術仕様が破綻する危険がある。

→ **統一テンプレート** と **圧縮仕様** により、  
AI間の認識差を最小化する。

---

## 📘 Core Concepts / 中核コンセプト

- **AIリレー（AI Relay）**  
  AI同士がバトンを渡しながらプロジェクトを継続する仕組み。

- **圧縮仕様（Compressed Spec）**  
  AIが誤解しないよう、構造だけを高密度にまとめた文書。

- **MASTER_HANDBOOK（完全版）**  
  人間・AIの両方が参照する百科事典的仕様書。

- **動的クレジット（Dynamic Credit）**  
  GitログやAI生成のクレジットにより、作者名を継承する仕組み。

- **多プロジェクト対応**  
  文学・技術・研究などジャンルを問わず管理できる。

## 🧭 Usage / 使い方

### 1. 新しいAIセッションを開始する前に  
- 対象プロジェクトの **HANDOFF_TEMPLATE.md** を開く  
- 最新の **COMPRESSED** と **SAMPLES** を貼り付ける  
- 必要に応じて **MASTER_HANDBOOK.md** を参照する

### 2. AIに渡すとき  
- テンプレートをそのままコピペ  
- コードブロックは1つにまとめる  
- AIに「前のAIの出力は参照しないで」と明記する

### 3. セッション終了後  
- SESSION_LOG を保存  
- COMPRESSED / SAMPLES に変更があれば更新  
- 必要なら MASTER_HANDBOOK.md も更新
