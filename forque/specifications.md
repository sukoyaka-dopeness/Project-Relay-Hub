# Specifications: Co-mit / Forque (開発仕様書：WHYを添えて)

## 1. 核心となる思想 (WHY)
* **「創作はクリシェ（再構築）である」:**
    * **WHY:** ゼロからの完全なオリジナルを強いることは、クリエイターに過剰な心理的負荷をかけます。「既存の何かから派生する」ことを「恥」ではなく「正当な系譜」として認めることで、コラボの心理的ハードルを下げ、創作の循環を加速させます。

## 2. UI用語の抽象化 (WHY)
技術用語を隠蔽し、クリエイターが直感的に操作できる名称を採用する。

| 技術用語 | 提案名称 | なぜこの名称か (WHY) |
| :--- | :--- | :--- |
| Issue | 壁打ち・お題 | 議論を事務作業ではなく、創作に向けた「思考のラリー」と定義するため。 |
| Pull Request | 取り込み依頼 | 技術的文脈から「他者の力を借りて作品を豊かにする」という創造的文脈へ変換するため。 |
| Push | 公開・保存 | 成果を記録・発表する能動的な行為であることを強調するため。 |
| Fork | 創作の種・派生 | 既存作品が次なる創作の出発点であるというポジティブな因果関係を示すため。 |

## 3. 信頼と安全性（心理的設計） (WHY)
* **歩幅（Pace）プロフィールの導入:**
    * **WHY:** 制作速度の不一致はコラボ解消の最大の火種です。事前に「自分の歩幅」を申告させることで期待値を調整し、トラブルを未然に防ぎます。
* **ポジティブな休止（アーカイブ）:**
    * **WHY:** 「飽きた」「逃げた」という負の感情を排除するため。創作をひとつの収穫物と捉え、次のサイクルへ進む儀式化をすることで、健全なフェードアウトを促します。
* **お断りのテンプレート:**
    * **WHY:** 拒絶による罪悪感をシステムが仲介することで、「機会のミスマッチ」という事務的な事実に置き換え、精神的負担をゼロにするため。

## 4. 収益と持続可能性 (WHY)
* **多角的な収益モデル:**
    * **WHY:** 収益源を多様化することで、売上のために「質」を犠牲にすることなく、コラボの「質」を重視する文化を維持するため。
* **自動収益分配:**
    * **WHY:** 著作権や分配金の交渉は挫折の最大の要因です。GitHubの貢献度グラフを応用した自動分配により、対人ストレスを排除します。

## 5. AI共存文化 (WHY)
* **AI使用率の可視化:**
    * **WHY:** AIの利用自体を悪とせず、正直な申告を奨励する設計。乖離があるユーザーを動的に分離することで、誠実なクリエイター同士の環境を保護・育成します。

## 6. 理想のチュートリアル・サンプル
* **系譜のデモ:** 1. 絵師Aが「森のキャラ」を描く（種を蒔く）。
    2. 小説家Bが影響を受けて「短編小説」を書く（インスパイア）。
    3. ミュージシャンCが「森のテーマ曲」を投稿する（コラボ）。
    4. 批評家Dがこれらを繋ぐ「批評」を投稿し、新たなブランチが生まれる。
    * **WHY:** 初回ユーザーが「自分の作品も誰かの種になり得る」という成功体験を直感的にイメージできるようにするため。

### MVP優先機能（Minimum Viable Product）
初期検証のための最小構成。心理的安全性と創作の系譜可視化の本質を最優先に検証する。  
まずはDiscord botで低コスト・低リスクにスタートし、ユーザー検証→イテレーションを繰り返す。

#### コアMVP（必須・最初に実装・検証対象）
- Pace申告（必須項目）とアイコン表示（閲覧時の判断材料としてカメ/ウサギ/ロケットなど視覚化）
- Wall-bouncing（テーマ）公開参加（オープンなテーマ投稿欄）
- Incorporation Request（コメント欄＋必要最低限のテンプレートメッセージ）
- Declination Templates（デフォルト複数用意＋プレビュー機能）
- ポートフォリオ必須（外部URL入力＋初回作品アップロード検証）
- 年齢・性別非表示徹底（World ID推奨で裏側認証、Orbバッジオプション）
- Discord bot実装（スラッシュコマンド中心: /pace, /theme, /request, /decline, /help, /start, /export-data）
  - 初回チュートリアル自動送信
  - 導入ハードル対策（招待説明、待機時間明記）
- 金銭授受なしモード（収益分配機能オフ）

#### MVP後期〜将来拡張（検証後優先順位で実装）
- 不可視誠実度スコアによるマッチング優先
- 自動収益分配（商用申告型＋ペナルティ）
- Git-like履歴（メタデータ＋外部ストレージ参照）
- 企業コンペ形式依頼機能
- Fediverse連携（v2.0 Federatedモード）
- FORQUE規格実装（貢献グラフ共有フォーマットなど）

**MVPのゴール**：  
心理的安全性を体感できる最小の創作ループ（テーマ投下 → Request → 系譜枝分かれ → 断り/継続）をDiscord内で成立させ、ユーザー数10〜50人規模でフィードバックを集める。
