# Future Challenges (未解決問題一覧)

本プラットフォーム「forque」（旧Co-mit / Forque）の実現に向け、優先的に解決・検証が必要な課題リスト。  
MVP（Discord botスタート）で可能な限り最小限対応し、残りを将来フェーズで深掘り。

## 1. コンテンツの真実性とストレージのジレンマ
* **メタデータ検証の負荷:** C2PA等の検証には原本ファイルへのアクセスが必要だが、サーバー側で全ファイルを保持するのはコストとプライバシーの観点で負担が大きい。
    * MVP対応：検証時の一時アップロード or ハッシュ値のみ保存＋外部API検証。
    * 将来：IPFS/Arweaveなどの分散ストレージ参照でプラットフォーム負担ゼロ化。
* **アナログ作品の証明:** プリント→再スキャンされたアナログ作品に対し、デジタル的に「制作の真正性」をどう保証するか。
    * MVP対応：他ユーザーからの「誠実性」評価（不可視スコア）で補完。C2PAはデジタル生まれ作品優先。
    * 将来：アナログ→デジタル変換時の手順ガイド + 評価蓄積による信頼バッジ。

## 2. 類似性検知と権利保護の自動化
* **自動検知アルゴリズム:** 画像・音声・テキストでの意図的改変（ノイズ付与、言い回し変更）による「持ち逃げ」をどの精度で自動排除できるか。
    * MVP対応：自動判断はNG（誤検知リスク高）。ユーザー投票（「homage」「疑わしい」フラグ）＋運営/モデレーターレビューで対応。
    * 将来：AIアシスト検知（CLIP類似度など）＋投票重み付けで精度向上。
* **誤検知の許容範囲:** オマージュと侵害の線引きをシステムでどう定義するか。
    * 将来：コミュニティガイドライン + 投票基準のRFC化。

## 3. 心理的安全性のための「角が立たない」UI
* **お断り・フェードアウトの自動化:** 罪悪感最小化テンプレートが相手を不快にさせず、正しく拒絶として伝わるか。
    * MVP対応：デフォルトテンプレート複数 + プレビュー機能。将来AIトーンチェック（オプション）。
    * 将来：A/Bテストで最適トーン検証。
* **リンク切れ・放置の扱い:** リンク切れバッジやスコア低下が創作を委縮させない配慮。
    * MVP対応：放置は「ポジティブアーカイブ」として扱い、ペナルティ軽め。

## 4. 収益分配の透明性と法的整合性
* **貢献度分配ロジック:** AI/複数協力者のプロジェクトでどこまで自動化、どこから人間合意か。
    * MVP対応：金銭授受なしモード（分配オフ）。商用申告型でスタート。
    * 将来：商用申告トリガー + 遡及分配（毎月/四半期グラフ保存）。弁護士レビュー済み規約必須。
* **スマートコントラクトの適用:** 自動実行時のプラットフォーム責任範囲。
    * 将来：FORQUE規格内でオープン定義 + 免責明記。

## 5. グローバル展開と用語の学習障壁
* **GitHub用語の抽象化:** 非技術者/多言語話者が「創作フロー」として直感操作できるUI表現。
    * MVP対応：日本語中心で「壁打ち」「取り込み依頼」など採用。英語版は後回し。
    * 将来：多言語対応 + チュートリアル強化。

## 6. 分散型/Fediverse移行時の課題（新規）
* **データ移行と互換性:** Discord bot時代 → Web → Federatedサーバーへのスムーズ移行。
    * MVP対応：/export-data コマンドでJSONエクスポート。ユーザー主導インポート。
    * 将来：ActivityPub拡張 or 独自プロトコルでサーバー間系譜同期。
* **プライバシーと信頼分散:** 中央DB依存を避けつつ、ユーザー主導のデータ管理。
    * MVP対応：最小データ保存 + オープンソースbotで自己運用可能。
* **初期ユーザー集めの壁:** マッチング成立にユーザー数が必要。
    * MVP対応：Discordコミュニティ提携 + 招待制スタート。

## 優先度まとめ（2026-03現在）
- 高優先（MVPで対応必須）：アナログ対応の評価ベース代替、自動検知回避（投票+レビュー）、収益オフモード、心理的安全性UI。
- 中優先（MVP後期〜v1.0）：FORQUE規格ドラフト、企業コンペ、モデレーター制度。
- 低優先（v2.0以降）：完全分散ストレージ、Fediverseフル連携、グローバル多言語。
