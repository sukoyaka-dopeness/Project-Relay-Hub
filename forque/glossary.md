## ブランチ・系譜

**本流 / Main Branch** `[UI: 幹]` `[規格: main-branch]`
正式に承認された中心ライン。本流の決定は参加者合意・作者判断・
代理人・継承ポリシー・本流なしの優先順位で行われる。
本質的に暫定的な概念であり、未来の創作文化では「本流なし」が
自然になる可能性がある。

**マルチ本流 / Multimain** `[規格: multimain]`
同一作品から派生した複数の本流が並列で存在するモード。
マルチバース型創作に対応する。コミュニティごとに異なる「本流」を
持つことが可能。

**本流なし / Mainless** `[規格: mainless]`
本流を定義しないモード。正統性を中央集権的に扱わない。
未来の創作文化（分散型・マルチバース型）への移行形態。

**孤立ブランチ / Isolated Branch**
他のブランチや本流に直接つながらず、独立して存在するフォーク。
「失敗」でも「拒絶」でもなく、「接続待ちの可能性」として扱う。
別の誰かが別のタイミングで手を伸ばすかもしれない。

**系譜 / Lineage** `[UI: 家系図を見る]` `[規格: lineage]`
作品の派生・合流・継承の全履歴。forqueの中核データ。
PC・タブレットでは全体ツリー表示、スマホでは親と直近の子のみ
の簡易表示。「森全体は遠くから見るもの」。

**派生元 / Origin** `[規格: origin]`
あるブランチが派生した元のブランチ。lineageに必ず記録される。

**合流 / Merge** `[規格: merge]`
複数のブランチが一つになること。異なるプロジェクト同士の合流も可能。
継承ポリシーが矛盾する場合は厳しい方が優先（自動統一）。

## アクション・フロー

**派生する / Fork** `[規格: fork]`
既存の作品を出発点として新しい作品・ブランチを作ること。
文化的に自然な行為として肯定される。

**つながり申請 / Pull Request** `[規格: pull-request]`
他者の作品に自分の作品を系譜としてつなげることを申請する行為。
承認・却下・無返答（90日）の3つの結果がある。

**公開する / Push** `[規格: push]`
作品を forque 上に公開する行為。公開前の作品にはつながり申請できない。

**投げかけ / Issue**
プロジェクトに向けた問いかけ・テーマ提案・議論の起点。
「壁打ち」「お題」とも呼ばれる。

**切断**
作者が既存の系譜から自作品を切断する操作。詳細設計は今後の課題。

## Pace（歩幅）

**Pace** `[UI: 歩幅]`
クリエイターの「だいたいのテンポ感」の自己申告。
返信の速さ・制作のペース・関与の密度の3軸を総合して選ぶ。
速さの優劣ではなく、等しく正当なテンポとして扱われる。

| アイコン | ラベル | 英語名 |
|---|---|---|
| 🐢 | ゆっくり | Slow |
| 🐇 | ふつう | Steady |
| 🚀 | 全力 | Sprint |
| 🫘 | 休眠（90日未ログイン） | Dormant |

**ユーザーPace**
アカウントに設定するデフォルトのPace。

**プロジェクトPace**
プロジェクトごとに上書き設定できるPace。
ユーザーPaceと異なる場合は両方表示＋説明文が出る。

**休眠 / Dormant**
90日未ログインでPaceアイコンが薄くなり🫘オーバーレイが重なった状態。
「失活」ではなく「次の季節への準備」として扱う。

## ポリシー・メタデータ

**継承ポリシー / Inheritance Policy** `[規格: inheritance-policy]`
作品がどのように派生・再利用されるかを定義する設定。
メインブランチ作成時に選択。派生ポリシーは「緩和のみ許可」（強化禁止）。

**本流ポリシー / Main Branch Policy** `[規格: main-branch-policy]`
本流の決定方法・条件・作者不在時の扱いを定義する設定。

**コピーレフト解放**
作者が設定した期間ログインがない場合に、本流ポリシーの権限のみ
解放される仕組み。著作権は保持される。期間は作者が自由に設定可能。
forque参考値は5年（強制力なし）。

**代理人 / co-maintainer** `[規格: co-maintainer]`
作者不在時に本流ポリシーの権限を引き継ぐ人物。
作者が事前に指定する。

**制作プロセス記録 / Process Log** `[規格: process-log]`
AI使用申告を含む制作プロセスのメタデータ記録。

**AI使用申告（プロセス層）**
AI関与を数値ではなくプロセスで申告する方式。
使用工程（企画/下書き/仕上げ/翻訳/その他）・
使用ツール（任意）・人間の主体性（主導/協働/補助的）を記録。
```json
{
  "generated-by": "human" | "ai" | "hybrid",
  "process": ["ideation", "finishing"],
  "human-agency": "lead" | "co-create" | "support"
}
```

## スコア・評価

**申告一致率**（旧: 誠実度スコア）
申告内容と実際の行動の一致をコミュニティの健全性の指標として
静かに観察する不可視スコア。存在は開示・仕組みは非開示。
つながり申請時の参考表示あり。トップページ優先表示なし。
完璧な誠実さの測定ではなく、不完全だけど誠実な試み。

**flag**
homage/盗作の疑いを申告する機能。回数制限・理由必須・
虚偽flagペナルティあり。MVP期は「議論ありバッジ」表示のみで判定なし。

**議論ありバッジ**
flagされたブランチに表示されるバッジ。MVP期の判定代替手段。

## ガバナンス

**陪審制**
homage/盗作の判定をコミュニティで行う仕組み。v1.0以降実装予定。
陪審員3〜5名（系譜2〜3段階離れ・当該ブランチ未参加）＋
AI参考人3種（類似度分析・メタデータ・判例）＋
人間参考人（当事者各1名・投票権なし）で構成。

**AI参考人**
陪審制において投票権を持たず情報提供のみを行うAI。
類似度分析AI・メタデータAI・判例AIの3種。

## UI・インターフェイス

**お断りテンプレート / Declination Template**
つながり申請を断る際に使う定型文。A（タイミング）・B（方向性）・
C（キャパシティ）・D（孤立ブランチ通知）の4種×日英。
主語は「私の状況」（相手の作品を評価しない）。

**「？」ヘルプ**
Pace変更画面などに設置された汎用ヘルプボタン。
Paceの3軸ガイド・用語説明（本glossary連携）を表示する。
説明を押しつけず、必要なときだけ参照できる設計。

**トースト通知**
Pace変更などの操作後に一時表示される軽量通知。
確認ダイアログの代替として使用。

## FORQUE規格

**FORQUE（規格）**
forqueのサービスが採用する創作プロセス規格。
他サービスへの展開・Fediverse連携を視野に入れた
オープンスタンダードとして設計される。

**FORQUEエクスポート**
forque上の自分のlineage・ポリシー設定・申告メタデータ等を
FORQUE規格のJSONで書き出す機能。
作品本体は外部サービスに存在するため対象外。
Fediverse移行・サービス間データ移動に使用。

## AIリレーセッション関連

**AIリレー**
複数のAIが段階的に仕様策定を引き継ぐ開発手法。
forqueの開発手法自体がforqueの系譜思想の実証例となっている。

**ハンドオフ / Handoff**
AIリレーセッション間の引き継ぎドキュメント。
context_state.jsonを正規リレー状態として扱う。
