# 1. コメント種別の定義

コメントは投稿時に種別を選択する。**種別は投稿後に変更できない**（後述）。

| 種別 | 表示名 | 到達範囲 | 備考 |
|---|---|---|---|
| 完全私的 | 「ひとりごと」 | 自分のみ | 外から一切見えない |
| クラスタ向け | 「つぶやき」 | 同クラスタ・距離減衰あり | 近いほど明瞭、遠いほど霧 |
| コンテンツ添付 | 「引用メモ」 | コンテンツに紐づく・距離減衰あり | 返信不可・集合非表示 |

---

# 2. 投稿後の種別変更ポリシー

**コメントは書いた時点の公開設定で固定する。**

- 「ひとりごと」→「つぶやき」への変換は、個別に明示的な操作が必要。
- 設定をグローバルに変更しても、**過去のコメントには遡及しない**。
- 理由：意図せず「ひとりごと」が公開される事故を防ぐため。安心感がサービスの根幹。

## コメントごとの種別選択

ブックマーク時のUI上で、コメントごとに「ひとりごと／つぶやき／引用メモ」を選択できる。
デフォルトはユーザー設定に従うが、投稿単位でオーバーライド可能。

---

# 3. デフォルト設定とオンボーディング

## オンボーディング猶予期間（検証対象仮説）

> 他サービスに前例のない設計。ベータで検証が必要。

- 登録後 **最初の30日間**、全コメントが「ひとりごと」に強制固定。
- 31日目以降に「つぶやきを許可する」スイッチが現れる。
- 意図：まず安全に慣れてから、自発的に勇気を出す設計。

**検証項目：**
- 猶予期間後に「つぶやき」に移行するユーザーの割合
- 猶予期間中の定着率
- 猶予期間の長さの最適値（30日が適切か）

# 4. コメント記法

長めのコメントを書く人（じっくり読んだ人）が書きやすい環境を整える。
装飾過多はMURMUR FIELDの静けさと合わないため、最小限の記法のみ対応する。

**対応記法：**
- `> 引用`（クォーテーション）
- `**太字**`
- 箇条書き（`-`）
- コードブロック（技術系クラスタ向け）

**非対応：** HTMLタグ全般

# 5. 連鎖フィードバック（数字を出さない通知設計）

## 原則
- 「◯人に届きました」という数値通知は出さない。
- 「誰に届いたか」も出さない。
- **気配と草原の現象だけを返す。**

## 3段階フィードバックモデル

### 第一段階：最初のブックマーク（発見者）
誰もブックマークしていないページを最初に保存したとき。

> 草が一本、静かに揺れる

カード上に細い草が一本だけそっと揺れるアニメーション。
「あなたが最初にここを見つけた」という意味。数字なし、言葉なし。

### 第二段階：最初の追っかけブックマーク
最も重要なフィードバック。はっきり伝える。

> 「あなたが見つけた場所に、誰かも来た気配があります」

草原に二本目の草が生えるアニメーション、または小さな波紋。**一度だけ表示。**

クラスタ外からの反応の場合：草の色や質感がわずかに異なる。
「なにか違う種類の気配」が伝わる程度の差分にとどめる。

### 第三段階：その後の追っかけ
マイページの隅に、持続的で静かなアイコン。
草が少しずつ増える、または淡い光が点灯したまま。頻繁に動かない。数を示さない。

**100人超えても表示は変わらない。** 仕様上、その情報は存在しないことにする。

# 6. 気配察知（通知設計）

## プル型通知
「気配を察知」ボタンを押したときだけ、気配が漂ってくる。
プッシュ型ではなく能動的に来た人だけが受け取れる設計。

## プッシュ通知
- デフォルト：**オフ**
- ユーザーが任意でオンにできる

## 段階的アンロック
サービス開始直後は通知感度を低く設定する。慣れてきた頃に感度が上がっていく。

> 草原に来たばかりの人は、まだ風の読み方を知らない。

アンロックの軸（未決・要検証）：利用期間 / ブックマーク数 / クラスタへの貢献度

---

# 7. 返信・集合の非生成

- 返信機能なし
- コメント集合ページなし
- 「◯件のコメント」表示なし
- 引用コメントは対象コンテンツに紐づくが、集合として見せない

# 9. UIの楽しみ（spec追記用メモ）

現行specに薄い部分として、以下をPhase設計に組み込む。

| 項目 | プラットフォーム | Phase目安 |
|---|---|---|
| キーバインドナビゲーション（J/Kで移動等） | PC | Phase 0 |
| ブックマーク時の触覚フィードバック（haptic） | スマートフォン | Phase 0 |
| 草原アニメーション（連鎖フィードバック） | 両方 | Phase 0 |
| 星座・菌糸ビュー（自クラスタの可視化） | 両方 | Phase 1 |
| 月次フィードバックのビジュアル表現 | 両方 | Phase 1 |

触る楽しみ・UIのリズムが離脱を防ぐ。Tumblrのキーバインド、RSSリーダーのサクサク感は「読む行為にリズムが生まれる」体験。スマートフォンのhapticはブックマークという行為を儀式化する。
